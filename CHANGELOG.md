# Changelog

All notable changes to `quecto` and `quecto-agent`, newest first.

- **`2026-07-22` â€” First official releases published.** [`quecto-v0.1.0`](https://github.com/adityak74/quecto/releases/tag/quecto-v0.1.0) and [`quecto-agent-v0.1.0`](https://github.com/adityak74/quecto/releases/tag/quecto-agent-v0.1.0) are now tagged and available as GitHub Releases, each with prebuilt `aarch64-apple-darwin` binaries attached. Additional platform builds will be added to these releases as CI cross-compilation lands.
- **`2026-07-22` â€” Multimodal image input support merged.** `quecto-agent` accepts a `--image` flag and serializes images as base64 into the request payload; verified end-to-end against a local vision model. (#25)
- **`2026-07-19` â€” Native evaluation harness (`quecto-eval`) shipped.** Created a new native Rust evaluation harness `quecto-eval` capable of parsing `eval.yaml` configurations to orchestrate agent test suites and evaluate their performance. It features a SQLite storage backend and polymorphic graders including `ScriptGrader` for bash validation, `TelemetryGrader` for threshold verification, and an extensible `LlmRubricGrader`. To power this, `quecto-agent` now natively supports dumping JSONL `TraceEvent` telemetry containing runtime, tool usage, and token tracking, which the evaluator validates against configured thresholds.
- **`2026-07-18` â€” Concurrent subagent spawning and monitoring.** `quecto-agent` can now run more than one subagent at once. `spawn_subagent` starts a subagent on a background thread and returns immediately with an id (up to `MAX_CONCURRENT_SUBAGENTS = 8` concurrent); `monitor_subagents` reports status, elapsed time, and recent activity for one or all spawned subagents; `cancel_subagent` stops one early via a cascaded cancellation token. Progress streams into a capped ring buffer through the existing `RunRecorder` trait, and background subagents render to a `NullRenderer` to avoid interleaving stdout across concurrent runs. `monitor_subagents` is always-allowed (read-only); `spawn_subagent`/`cancel_subagent` follow the same policy as the existing background-process tools. The original synchronous `invoke_subagent` tool is unchanged.
- **`2026-07-18` â€” Native Anthropic Claude API provider.** `quecto-agent` can now talk to Claude models directly via the Anthropic Messages API (`/v1/messages`), alongside the existing OpenAI-compatible path. Select it with `--provider anthropic`, `provider = "anthropic"` in a flavor file, or `QUECTO_PROVIDER=anthropic` (same `QUECTO_API_KEY` for both providers). Handles Anthropic's wire format end-to-end â€” `system`-role extraction, `tool_use`/`tool_result` content blocks, `x-api-key`/`anthropic-version` auth, required `max_tokens` (configurable via `--max-tokens` / `max_tokens` / `QUECTO_MAX_TOKENS`) â€” and maps `reasoning_mode` onto Anthropic's `thinking.budget_tokens`. No streaming or other beta features yet.
- **`2026-07-18` â€” Reasoning & Mermaid chat support shipped.** `quecto-agent chat` now supports session-scoped reasoning controls via `/reasoning`, persists that session default across `resume`, renders assistant markdown in interactive TTY sessions, and renders fenced Mermaid diagrams inline with Unicode box drawing via `merman`. Non-TTY output stays raw/plain markdown, and non-renderable Mermaid fences fall back to their original source text.
- **`2026-07-16` â€” Agent core tightened.** Merged a hygiene refactor that removed **126 lines** from `quecto-agent` with no behavior change: deduped chat command dispatch across TTY and piped-stdin REPL paths, deleted redundant `canonical_json`, removed dead code, wired up `/commands`, and named the repeat-guard limit. Live cmux UAT covered both REPL branches, a real local-Ollama chat turn, and `/undo`.
- **`2026-07-16` â€” Chat UI Polish & Tool Enhancements.** Dramatically improved the terminal UI for `quecto-agent chat`. Tool summaries are now highly verbose (e.g., `diff (449 lines)`, `exited 1`, `created src/a.rs (12 lines)`) instead of terse single words, and `.qkb` note tools explicitly enforce their directory in both descriptions and output to prevent model hallucination.
- **`2026-07-16` â€” Chat UX & Agent Delegation shipped.** Upgraded the chat REPL to a `crossterm` event loop with native Bracketed Paste support. Added tracking for background processes (`start_background_process`, `kill_background_process`), a new `invoke_subagent` tool for agent delegation, and project-local `.qkb` Markdown notes.
- **`2026-07-15` â€” Evaluation suite shipped.** A 10-task TerminalBench-style smoke suite (`evals/smoke/`) with deterministic `verify.sh` verifiers and a [Harbor](https://harborframework.com) adapter (`evals/harbor/quecto_agent.py`) for the full 89-task Terminal-Bench 2.x benchmark. Run `./evals/run_evals.sh` â€” no API key needed.
- **`2026-07-15` â€” OpenTelemetry (OTEL) support.** Gated behind the `otel` feature flag, `quecto-agent` now supports end-to-end tracing for run loops, steps, tool dispatches (with argument sanitization), and model completions â€” including parsed reasoning/thinking traces.
- **`2026-07-14` â€” Bug-fix release.** Seven issues from a post-UAT audit fixed: CRLF patch compatibility (with double-conversion guard), `take_last_change` DB error propagation, millisecond timestamps, `record_message` transaction safety, `/status` session-specific querying, `/context` character count, and REPL UX polish (aliases, `--version`, clear confirmation). 183 tests, 0 failures.
- **`2026-07-12` â€” `quecto-agent` shipped (M1â€“M7b).** The full coding agent â€” tool use, editing under approval, sandbox denylist, verification gates, session persistence (resume/undo/diff), and manifest flavors with trust-on-first-use â€” is complete and merged to `main`.
- **`2026-07-12` â€” UAT accepted.** 41 black-box scenarios across CLI, chat, tools, persistence, and flavors run against a live model: 34 pass, 7 minor polish partials, **0 failures, 0 blocking defects**. See [`docs/UAT-report.md`](docs/UAT-report.md).
- **`2026-07-10` â€” Core crate landed.** The full `quecto` core: four-function library API, streaming with SSE + non-SSE fallback, and a one-shot / REPL / `--init` CLI. 24 tests, clippy-clean, two dependencies.
- **`2026-07-10` â€” Size-optimized build.** A tuned release profile ships both binaries statically-linked, no runtime: the core at **~1.3 MB**, the agent at **~3.5 MB**.
